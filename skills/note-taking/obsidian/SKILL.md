# Obsidian Vault

Use this skill for filesystem-first Obsidian vault work: reading notes, listing notes, searching note files, creating notes, appending content, and adding wikilinks.

## Arijit's Vault Contract

Obsidian is Arijit's central source of truth for projects, knowledge, decisions, raw captures, synthesized notes, and operating memory.

Known vault locations:

```text
Mac vault container: /Users/arijitchowdhury/Dropbox/AI-Development/Personal/Obsidian-Vault
Mac active vault root: /Users/arijitchowdhury/Dropbox/AI-Development/Personal/Obsidian-Vault/MyOS
Hermes VPS mirror: /opt/data/knowledge/obsidian/MyOS
Default Hermes write inbox: /opt/data/knowledge/obsidian/MyOS/Chowmes-Inbox
```

The active Obsidian vault root is `MyOS` because it contains `.obsidian/`.

On live ChowMes/Hermes, resolve the vault path from `OBSIDIAN_VAULT_PATH`. The expected value is:

```text
/opt/data/knowledge/obsidian/MyOS
```

If `OBSIDIAN_VAULT_PATH` is unavailable in a tool context, use `/opt/data/knowledge/obsidian/MyOS` on the VPS and the Mac active vault root above on the local machine.

## Sync Boundary

The VPS mirror is a filesystem mirror for Hermes. It is not Obsidian Sync itself.

- Read from the VPS mirror during live Hermes work.
- Write generated notes to `Chowmes-Inbox/` by default unless the user names a specific target note or project folder.
- Do not edit `.obsidian/` settings, workspace files, plugin configs, or app metadata.
- Use the ChowMes sync scripts to push the local vault to Hermes and pull Hermes-generated inbox notes back into the Mac vault.

## Read A Note

Use `read_file` with the resolved absolute path to the note. Prefer this over `cat` because it provides line numbers and pagination.

## List Notes

Use `search_files` with `target: "files"` and the resolved vault path. Prefer this over `find` or `ls`.

- To list all markdown notes, use `pattern: "*.md"` under the vault path.
- To list a subfolder, search under that subfolder's absolute path.

## Search

Use `search_files` for both filename and content searches. Prefer this over `grep`, `find`, or `ls`.

- For filenames, use `search_files` with `target: "files"` and a filename `pattern`.
- For note contents, use `search_files` with `target: "content"`, the content regex as `pattern`, and `file_glob: "*.md"` when you want to restrict matches to markdown notes.

## Create A Note

Use `write_file` with the resolved absolute path and the full markdown content. Prefer this over shell heredocs or `echo` because it avoids shell quoting issues and returns structured results.

Default new-note location:

```text
<vault-root>/Chowmes-Inbox/YYYY-MM-DD-title-slug.md
```

When writing notes, include:

- a clear title
- creation date
- source or triggering request when useful
- links to related notes using wikilinks
- a short status block when the note is a project, decision, SOP, or research artifact

## Append To A Note

Prefer a native file-tool workflow when it is not awkward:

- Read the target note with `read_file`.
- Use `patch` for an anchored append when there is stable context, such as adding a section after an existing heading or appending before a known trailing block.
- Use `write_file` when rewriting the whole note is clearer than constructing a fragile patch.

For a simple append with no stable context, `terminal` is acceptable if it is the clearest safe option.

## Targeted Edits

Use `patch` for focused note changes when the current content gives you stable context. Prefer this over shell text rewriting.

Do not overwrite a long note unless the user explicitly asks or the file is generated by Hermes and the rewrite is clearly bounded.

## Wikilinks

Obsidian links notes with `[[Note Name]]` syntax. When creating notes, use these to link related content.

Prefer stable project and topic links such as:

- `[[Projects]]`
- `[[Knowledge]]`
- `[[DecisionLog]]`
- `[[Architecture]]`
- `[[Chowmes]]`

## Safety

- Do not write secrets, API keys, passwords, tokens, recovery codes, or private keys into the vault.
- Do not ingest or summarize private personal notes unless the user asks for that specific folder or file.
- Preserve source evidence and distinguish fact, inference, recommendation, and open question.
- Keep generated knowledge in markdown that remains useful outside any one AI tool.
